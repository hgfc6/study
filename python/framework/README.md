# Python 常用框架总览

这个目录用于整理 Python Web 开发和常见框架知识。

## 当前框架

| 框架 | 定位 | 适合场景 | 入口 |
| --- | --- | --- | --- |
| Django | 全功能 Web 框架 | 后台管理、内容站、企业管理系统 | [Django](./Django/README.md) |
| Flask | 轻量级 Web 框架 | 小型 Web 服务、教学 Demo、灵活组装项目 | [Flask](./Flask/README.md) |
| FastAPI | 现代 API 框架 | REST API、异步接口、自动接口文档 | [FastAPI](./FastAPI/README.md) |

## 三个框架怎么选

1. Django
   - 优点：自带 ORM、模板、后台管理、认证、路由、迁移。
   - 代价：框架约定较多，项目结构更重。
   - 建议：想快速做后台系统或完整网站时优先学。
2. Flask
   - 优点：轻量、直观、学习曲线低。
   - 代价：数据库、登录、表单等能力通常需要自己选择扩展。
   - 建议：适合理解 Web 基础概念，也适合做小型服务。
3. FastAPI
   - 优点：类型标注友好、自动生成 OpenAPI 文档、天然支持 async。
   - 代价：需要理解类型标注、Pydantic、ASGI。
   - 建议：做前后端分离 API 或微服务时很适合。

## Web 框架通用知识点

1. 路由：把 URL 和处理函数对应起来。
2. 请求：读取查询参数、路径参数、请求体、请求头。
3. 响应：返回 HTML、JSON、文件或重定向。
4. 模板：把后端数据渲染成 HTML 页面。
5. 数据库：通过 ORM 或 SQL 存取数据。
6. 表单校验：检查用户输入是否合法。
7. 登录认证：识别当前用户是谁。
8. 权限控制：判断用户能否访问某个资源。
9. 中间件：在请求前后统一处理逻辑。
10. 部署：开发服务器不能直接当生产服务器使用。

## 建议学习顺序

1. 先用 Flask 理解路由、请求、响应、模板。
2. 再用 Django 理解完整项目结构、ORM、后台管理。
3. 最后学习 FastAPI，掌握类型标注、接口文档和异步 API。
