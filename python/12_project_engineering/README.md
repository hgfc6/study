## 为什么要学工程化

基础语法解决的是“代码能不能写出来”，工程化解决的是“代码能不能维护、配置、排查和复用”。

常见场景：

1. 程序需要不同环境的配置，例如开发环境和生产环境。
2. 程序需要记录运行过程，方便定位错误。
3. 脚本需要接收命令行参数，避免每次都改源码。
4. 项目变大后，需要清晰的目录结构和依赖管理。
